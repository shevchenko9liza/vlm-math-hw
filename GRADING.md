# Оценивание — 10 баллов

Основная оценка ставится за корректную реализацию пайплайна, а не за качество большой модели.

| Часть | Баллы | Что оценивается |
|---|---:|---|
| `dataset.py` | **1.5** | чтение `manifest.jsonl`, загрузка изображений, фильтрация split/subject, обработка вопросов и ответов |
| `processor.py` | **2.0** | обработка изображений, prompt, image tokens, `input_ids`, `attention_mask`, `labels`, `collate_fn` |
| `model.py` | **2.0** | adapter, вставка visual embeddings, объединение visual/text embeddings, forward/generate |
| `train.py` | **1.0** | корректный smoke-training loop, finite loss, gradient accumulation, сохранение adapter/checkpoint |
| `benchmark.py` | **1.0** | evaluation prompt, парсинг ответа, подсчёт accuracy overall/by subject |
| Public tests | **1.0** | решение проходит `pytest -q tests_public` без изменения публичных тестов |
| `report.md` | **1.0** | описано, что реализовано, как запускалось, какие тесты пройдены, выводы и ошибки модели |

Итого: **10 баллов**.

## Дополнительные условия

- Если код не запускается из-за синтаксических ошибок, максимум за задание — **3 балла**.
- Если не сдана ссылка на репозиторий, работа не проверяется.
- Менять `tests_public/` ради прохождения тестов нельзя.
- Toy-набор `assets/toy_math_vqa/` используется только для проверки работоспособности пайплайна; качество модели по нему не оценивается.
- MathVista относится к расширенной проверке качества / бонусной части и не требуется для получения базовых 10 баллов.
- Большие датасеты, MathVista и checkpoints не нужно коммитить в репозиторий.

## Бонус / расширенная часть

Бонусные баллы могут быть начислены отдельно по решению преподавателя за:

- запуск quality evaluation на MathVista testmini;
- аккуратный анализ ошибок модели;
- улучшение качества относительно простого baseline;
- воспроизводимый эксперимент с указанием config, seed и ресурсов.
